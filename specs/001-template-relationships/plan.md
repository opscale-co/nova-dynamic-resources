# Implementation Plan: Template Relationships

**Branch**: `001-template-relationships` | **Date**: 2026-05-03 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-template-relationships/spec.md`

## Summary

Add a `Relationship` entity per Template (parallel to `Field` and `Action`) that declares typed relations (BelongsTo / HasOne / HasMany) between two Templates. Foreign keys live in the existing `data` JSON column of source/target records — no pivot table. A new Eloquent trait resolves relation method calls dynamically via `__call`/`isRelation`, returning custom `Json*` Relation subclasses that query `data->{fk}`. A `RenderRelationship` action mirrors `RenderField` and resolves the related Nova Resource via the existing `app('dynamic-{uri_key}')` binding. The legacy `parent` and `children` field types are removed from the config registry.

## Technical Context

**Language/Version**: PHP 8.3
**Primary Dependencies**: Laravel 11, Laravel Nova 5, Eloquent ORM, Pest 3, Orchestra Testbench, Laravel Dusk, opscale-co/actions, opscale-co/validations
**Storage**: SQLite (testbench), MySQL/MariaDB/Postgres in production. JSON path queries via Eloquent `data->key` syntax.
**Testing**: Pest (unit + feature) + Laravel Dusk (browser) under `tests/Unit`, `tests/Feature`, `tests/Browser`.
**Target Platform**: Composer package distributed via Packagist; consumers run on Laravel 11+.
**Project Type**: Laravel Nova package (single repo).
**Performance Goals**: Relationship eager loading must not exceed 2 SQL queries (1 source + 1 batched target) per relationship name regardless of source row count.
**Constraints**: Backward-incompatible removal of `parent`/`children` field types is acceptable (pre-1.0). No pivot table introduced. JSON-path queries must work on SQLite + MySQL + Postgres.
**Scale/Scope**: A typical Template carries 0–10 Relationships; a typical record set is 100–10,000 rows. The dynamic Resource registry (`'dynamic-{uri_key}'`) already scales linearly with template count.

## Phase 1 — Design Artifacts

### Data model (`data-model.md` summary inline)

`dynamic_resources_relationships`:

| col | type | notes |
|---|---|---|
| id | ULID PK | |
| template_id | ULID FK → templates | cascade |
| name | string | unique per template_id |
| label | string | |
| cardinality | enum(BelongsTo, HasOne, HasMany) | |
| related_template_id | ULID FK → templates | required; cascade |
| foreign_key | string | regex `^[a-z][a-z0-9_]*$` |
| inverse_name | string nullable | |
| required | bool | default false |
| rules | json nullable | |
| config | json nullable | |
| timestamps + soft_deletes | | |

Indexes: `(template_id)`, `(related_template_id, cardinality)`, unique `(template_id, name)` (composite).

### Eloquent contract

`HasDynamicRelationships` trait exposes:

- `__call($method, $args)` — falls back to relation lookup, then `parent::__call`.
- `isRelation($key)` — returns true if a Relationship row matches.
- `resolveDynamicRelationship($name)` — instance-cached lookup; checks owner-side `relationships` first, then inverse via `inverse_name`.
- `buildDynamicRelation(Relationship $r)` — switches on cardinality and returns one of `JsonBelongsTo` / `JsonHasOne` / `JsonHasMany`.

Each `Json*` Relation subclass overrides `addConstraints`, `addEagerConstraints`, `initRelation`, `match`, `getResults`, building its query via Eloquent's `data->{fk}` JSON path.

### Nova contract

`RenderRelationship::run([...])` resolves: target Resource via `app('dynamic-'.$rel->relatedTemplate->uri_key)`, then `match` on cardinality to `Laravel\Nova\Fields\BelongsTo|HasOne|HasMany::make($label, $name, $resourceClass)`. Applies `withMeta($rel->config)` and passes `rules`.

### Configuration registry

`config/nova-dynamic-resources.php` removes top-level `'parent'` and `'children'` entries and adds:

```php
'relationships' => [
    'belongs_to' => ['field' => Laravel\Nova\Fields\BelongsTo::class, 'rules' => ['nullable']],
    'has_one'    => ['field' => Laravel\Nova\Fields\HasOne::class,    'rules' => []],
    'has_many'   => ['field' => Laravel\Nova\Fields\HasMany::class,   'rules' => []],
],
```
